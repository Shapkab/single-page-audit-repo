# Security Policy

## Supported Versions

Security updates are provided for the latest `main` branch state.

## Reporting a Vulnerability

Please report vulnerabilities privately and do not open public issues.

Primary channel:
- GitHub private vulnerability reporting (Security Advisories).

When reporting, include:
- Affected component/file
- Reproduction steps or proof of concept
- Impact assessment
- Suggested remediation (if available)

You can expect an initial response within 5 business days.
